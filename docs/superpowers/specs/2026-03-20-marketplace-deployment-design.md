# Marketplace Deployment Design Spec

## Overview

Design for distributing the cost-guardrail Claude Code plugin via a public GitHub marketplace repo. Users install with a single `curl | bash` command that handles cloning, marketplace registration, plugin installation, and settings.json hook setup.

## Decisions

| Decision | Choice |
|----------|--------|
| Distribution model | Public GitHub marketplace repo |
| Marketplace repo name | `gonsoomoon-ml/bedrock-cost-guardrail` |
| Plugin name | `bedrock-cost-guardrail` (renamed from `cost-guardrail`) |
| Install UX | Single `curl \| bash` command via `install.sh` |
| Release mechanism | Manual `scripts/release.sh` (simple file copy) |
| Config strategy | Distribution defaults ($50 threshold), not source repo values |
| Marketplace README language | English |
| Command names | `/cost-status`, `/cost-config` (no `bedrock-` prefix on commands, plugin prefix already contains it) |

## Two-Repo Model

| | Source Repo | Marketplace Repo |
|---|---|---|
| Repo name | `cost-guardrail-claude-code-bedrock` | `bedrock-cost-guardrail` |
| Purpose | Development, docs, design specs | Public distribution |
| Plugin name | `bedrock-cost-guardrail` | `bedrock-cost-guardrail` |
| Audience | Contributors | End users |
| README language | Korean | English |

The source repo cannot serve as a marketplace directly because a repo cannot have both `plugin.json` and `marketplace.json` at the root level.

## install.sh

Lives in the marketplace repo. Users run:

```bash
curl -sSL https://raw.githubusercontent.com/gonsoomoon-ml/bedrock-cost-guardrail/main/install.sh | bash
```

### Behavior

1. **Check prerequisites** — Verify `claude`, `jq`, `aws`, `bc` are available. Exit with clear error if any missing.
2. **Clone or update** — Clone repo to `~/.claude/plugins/bedrock-cost-guardrail/`. If directory already exists, run `git pull` to update.
3. **Register marketplace** — `claude plugin marketplace add ~/.claude/plugins/bedrock-cost-guardrail`
4. **Install plugin** — `claude plugin install bedrock-cost-guardrail@bedrock-cost-guardrail`
5. **Patch settings.json** — Add SessionStart and UserPromptSubmit hooks to `~/.claude/settings.json`:
   - If file doesn't exist, create it with hooks
   - If file exists, merge hooks into existing config using `jq`
   - If hooks already exist (re-install), update command paths
6. **Print success** — Show installed status and verify commands

### settings.json Hook Entries

The installer resolves `PLUGIN_PATH` by using the cloned marketplace repo's plugin directory directly: `~/.claude/plugins/bedrock-cost-guardrail/plugins/bedrock-cost-guardrail`. This avoids depending on the plugin cache path (which varies by marketplace implementation) and uses the known, stable clone location instead.

```jsonc
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash <PLUGIN_PATH>/hooks/check-cost.sh --event session_start",
            "timeout": 60
          }
        ]
      }
    ],
    "UserPromptSubmit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash <PLUGIN_PATH>/hooks/check-cost.sh --event prompt_submit",
            "timeout": 30
          }
        ]
      }
    ]
  }
}
```

### Error Handling

The installer fails closed (opposite of the plugin's fail-open policy):
- Missing prerequisite → error message listing what to install, exit 1
- Git clone fails → error message, exit 1
- Marketplace add fails → error message, exit 1
- Plugin install fails → error message, exit 1
- settings.json patch fails → error message with manual instructions, exit 1

No step is silently skipped.

## scripts/release.sh (Source Repo)

Lives in the source repo at `scripts/release.sh`. Run manually when releasing a new version.

### Usage

```bash
bash scripts/release.sh /path/to/bedrock-cost-guardrail
```

### Behavior

1. **Validate target** — Check target directory has `.claude-plugin/marketplace.json`. Exit if not found.
2. **Clean target** — Remove existing `plugins/bedrock-cost-guardrail/` directory in target.
3. **Copy plugin files** from source repo to `plugins/bedrock-cost-guardrail/` in target:
   - `.claude-plugin/plugin.json`
   - `commands/cost-status.md`
   - `commands/cost-config.md`
   - `hooks/hooks.json`
   - `hooks/check-cost.sh`
   - `skills/cost-awareness/SKILL.md`
4. **Generate distribution config.json** — Write hardcoded safe defaults (see Distribution config.json above), not the source repo's working config.
5. **Print summary** — List files copied and remind to commit+push the marketplace repo.

### What is NOT copied

- `CLAUDE.md` — source repo development guidance
- `README.md` — Korean, source-repo-specific
- `LICENSE` — marketplace repo has its own
- `docs/` — design specs, beginners guide
- `img/` — screenshots for source repo README
- `scripts/` — release tooling stays in source repo

## Changes to Source Repo

1. **Rename plugin** — Update `.claude-plugin/plugin.json` name from `cost-guardrail` to `bedrock-cost-guardrail`
2. **Add release script** — Create `scripts/release.sh`
3. **Update CLAUDE.md** — Note the rename and add release workflow reference
4. **Update README.md** — Update command prefixes from `/cost-guardrail:` to `/bedrock-cost-guardrail:`
5. **Fix check-cost.sh log_group fallback** — Change default from `aws/bedrock/model-invocations` to `bedrock/model-invocations` (line 44)

## User-Facing Command Reference

After installation, users have:

| Command | Purpose |
|---------|---------|
| `/bedrock-cost-guardrail:cost-status` | Check current estimated cost |
| `/bedrock-cost-guardrail:cost-config show` | View settings |
| `/bedrock-cost-guardrail:cost-config set <key> <value>` | Modify settings |

## Uninstall

Not automated in v1. Manual steps:
1. Remove hooks from `~/.claude/settings.json`
2. `claude plugin uninstall bedrock-cost-guardrail` (if supported)
3. Remove marketplace: `claude plugin marketplace remove bedrock-cost-guardrail` (if supported)
4. Remove `~/.claude/plugins/bedrock-cost-guardrail/`
